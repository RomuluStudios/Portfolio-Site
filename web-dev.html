GradePilot - Intelligent Academic Priority System (V1.0 Test Build)

Link To Github Repo

GradePilot is a task prioritization system designed to help students manage academic workload across multiple courses, deadlines, and time constraints. Instead of presenting users with a static to-do list, the system builds a structured model of a student’s week using work schedules, class timetables, assignments, due dates, estimated effort, and self-reported difficulty levels. At the core of the application is a priority engine that evaluates each task using multiple weighted factors, including urgency, academic impact, workload size, available time, and course risk. It also considers real-world constraints such as overlapping commitments, free time distribution, and task completion feasibility within upcoming deadlines. The result is a ranked set of recommendations that dynamically adjusts based on what the student can realistically accomplish. Each recommendation includes a clear action, supporting reasoning, and an estimated time allocation for the current day. To reduce cognitive overload, the interface separates immediate focus (“Do This Now”) from secondary recommendations, allowing users to either act immediately or explore deeper breakdowns of how each recommendation was generated.

Feature Walkthrough

Adding Schedule

Users are able to add their work schedule, choose its frequency (weekly, bi-weekly or monthly) and choose whether that is a permanent or changing schedule.


In addition to the users work schedule, they can also add their class schedule as well as any one-off appointments they have.

Importing Courses

Users are able to upload course CSI documents, and autofill course information such as grading categories (assignments, labs, etc.) as well as the specific items within those categories. This uses Paddle OCR through a cloud based server, and uploaded images are deleted after parsing and user confirmation. This allows users to ensure OCR accuracy before importing information, in addition to the ability to edit courses themselves from the course screen.

Course Editing

The course screen allows users to edit assignments, categories or entire courses. Users are able to add/update due dates, their estimated difficulty of the assignments, as well as their estimated time to completion. Courses are sorted by semester, allowing easy navigation if users utilize GradePilot across multiple semesters.

Priority Engine (Home Screen)

The Priority Engine is the core system that determines what a student should work on at any given time. It builds a structured model of workload using course schedules, work shifts, class timetables, assignments, due dates, estimated completion time, and difficulty ratings, combining everything into a unified view of time and obligations.

Each task is evaluated using multiple weighted factors:

  • Urgency based on how close the due date is
  • Academic impact based on assignment weight and course structure
  • Time feasibility based on available free time before the deadline
  • Effort size using estimated completion time and difficulty
  • Course risk based on current grade performance and progression
  • Workload balance to avoid overloading a single day

Instead of relying on a single rule such as due date alone, the engine calculates a combined priority score that reflects both urgency and real-world feasibility. This ensures recommendations are not only important, but actually achievable within the student’s available time.

The final output is a ranked list of recommendations, including a primary “Do This Now” task and supporting secondary actions. Each recommendation includes a suggested work duration and a reason explaining why it was selected.

To improve usability, the system applies safeguards that prevent unrealistic scheduling, such as overloading a single day, prioritizing tasks without enough context, or ignoring limited available time.
